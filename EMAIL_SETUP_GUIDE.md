# Order Confirmation Email Setup Guide

## Overview

Your Klox Systems application now has a complete order confirmation email system! When a customer places an order, they automatically receive a detailed email with:

✅ Order details and ID
✅ All items purchased with quantities and prices
✅ Full price breakdown (subtotal, tax, shipping)
✅ Shipping and billing addresses
✅ Payment method and confirmation status
✅ Estimated delivery date
✅ Next steps and customer information

## Available Email Services

The application supports **3 email delivery methods** (pick one):

### 1. **EmailJS** (Recommended for Quick Setup)
- ✅ No backend required
- ✅ Free tier: 200 emails/month
- ✅ Setup takes 5 minutes
- ✅ Works entirely from frontend
- ❌ Limited to 200/month on free plan

### 2. **Firebase Cloud Function** (Recommended for Production)
- ✅ Serverless, scales automatically
- ✅ Integrates with Firebase
- ✅ Can use SendGrid, Mailgun, Gmail
- ✅ No cost for small volume
- ⚠️ Requires Cloud Function deployment

### 3. **Custom Backend API** (For Existing Backend)
- ✅ Use your own server
- ✅ Full control
- ✅ Can integrate with any email provider
- ❌ Requires backend infrastructure

## Setup Instructions

### Option 1: EmailJS (Easiest)

#### Step 1: Create EmailJS Account
1. Go to https://www.emailjs.com/
2. Click "Sign Up" and create a free account
3. Verify your email

#### Step 2: Create Email Service
1. In EmailJS dashboard, go to "Email Services" 
2. Click "Create New Service"
3. Choose "Gmail" or "SMTP" (Gmail is easiest)
4. Follow the connection steps
5. Note your **Service ID** (looks like `service_xxxxx`)

#### Step 3: Create Email Template
1. Go to "Email Templates"
2. Click "Create New Template"
3. Set Template ID to: `template_order_confirmation`
4. Use this template content:

```html
{{order_html}}
```

This will use the HTML generated by our system.

#### Step 4: Get Your API Key
1. Go to "Account" in top menu
2. Under "API Keys", copy your **Public Key**
3. Keep this private!

#### Step 5: Configure in Your App

Edit [js/email.js](js/email.js) and update the EmailJS config:

```javascript
emailjs: {
  serviceId: 'service_XXXXX',        // From your EmailJS account
  templateId: 'template_order_confirmation',
  apiKey: 'YOUR_EMAILJS_PUBLIC_KEY', // Your public key
  enabled: true,  // ⚠️ Change this to true
},
```

#### Step 6: Test It
1. Refresh your app
2. Open browser console (F12)
3. Run: `EmailService.config.emailjs.enabled ? 'Enabled' : 'Disabled'`
4. Should show: `Enabled`

✅ **Done!** Orders will now send emails.

### Option 2: Firebase Cloud Function

#### Step 1: Deploy Cloud Function
Create a new file in Firebase Cloud Functions directory:

**`functions/sendOrderConfirmationEmail/index.js`**
```javascript
const functions = require('firebase-functions');
const nodemailer = require('nodemailer');

// Configure your email service (SendGrid, Mailgun, Gmail, etc.)
const transporter = nodemailer.createTransport({
  service: 'gmail',
  auth: {
    user: process.env.EMAIL_USER,
    pass: process.env.EMAIL_PASSWORD,
  }
});

exports.sendOrderConfirmationEmail = functions.https.onRequest(async (req, res) => {
  if (req.method !== 'POST') {
    return res.status(405).send('Method Not Allowed');
  }

  try {
    const { to, subject, htmlContent } = req.body;

    if (!to || !subject || !htmlContent) {
      return res.status(400).send('Missing required fields');
    }

    const mailOptions = {
      from: process.env.EMAIL_USER,
      to: to,
      subject: subject,
      html: htmlContent,
    };

    await transporter.sendMail(mailOptions);
    res.json({ success: true, message: 'Email sent' });
  } catch (error) {
    console.error('Error sending email:', error);
    res.status(500).json({ success: false, error: error.message });
  }
});
```

#### Step 2: Set Environment Variables
```bash
firebase functions:config:set email.user="your-email@gmail.com"
firebase functions:config:set email.password="your-app-password"
```

#### Step 3: Deploy
```bash
firebase deploy --only functions
```

#### Step 4: Get Your Function URL
After deployment, copy the HTTPS trigger URL from Firebase Console.

#### Step 5: Configure in Your App

Edit [js/email.js](js/email.js):

```javascript
firebaseCloudFunction: {
  url: 'https://us-central1-YOUR-PROJECT.cloudfunctions.net/sendOrderConfirmationEmail',
  enabled: true,  // ⚠️ Change this to true
},
```

### Option 3: Custom Backend API

#### Step 1: Create Backend Endpoint
Your backend should have a POST endpoint like:
```
POST https://your-backend.com/api/send-email
```

Expecting payload:
```javascript
{
  to: "customer@example.com",
  subject: "Order Confirmation - ORD-xxx",
  htmlContent: "<html>...</html>",
  orderId: "ORD-xxx",
  customerName: "John Doe",
  orderTotal: 5999.00
}
```

Response should be:
```javascript
{
  success: true,
  messageId: "xxx",
  timestamp: "2026-02-20T10:30:00Z"
}
```

#### Step 2: Configure in Your App

Edit [js/email.js](js/email.js):

```javascript
customBackend: {
  url: 'https://your-backend.com/api/send-email',
  enabled: true,  // ⚠️ Change this to true
},
```

## How It Works

### Order Placement Flow

```
Customer places order
         ↓
Payment processing
         ↓
Order created
         ↓
EmailService.sendOrderConfirmation() called
         ↓
Tries available methods (Firebase → EmailJS → Custom)
         ↓
Email sent successfully
         ↓
Customer receives confirmation in inbox
```

### Email Content

The confirmation email includes:

**Header**
- ✓ Order Confirmed badge
- Order ID

**Order Details**
- Order date & time
- Order status
- Estimated delivery date

**Items Table**
- Product name & quantity
- Unit price
- Total per item

**Price Summary**
- Subtotal
- Tax calculation
- Shipping cost
- **Total Amount**

**Payment Information**
- Payment method used
- Payment status (Confirmed)
- Transaction ID

**Addresses**
- Shipping address (full details)
- Billing address (if different)

**Next Steps**
- Order verification timeline
- When tracking will be sent
- What to expect

**Customer Info**
- Name
- Email
- Phone number

## Troubleshooting

### "No email service configured"
**Problem:** You see this warning in console
**Solution:** Enable at least one email service:
- Set `enabled: true` in config
- Make sure API keys/URLs are correct

### Email not received
**Check:**
1. Is email service enabled? `EmailService.config.emailjs.enabled` should be `true`
2. Check browser console for errors
3. Check spam/promotions folder
4. Verify email address is correct

### "EmailJS not loaded"
**Problem:** `typeof emailjs === 'undefined'`
**Solution:** 
1. EmailJS script failed to load from CDN
2. Check network tab in DevTools
3. Check that `emailjs.enabled` config is correct

### CORS errors
**Problem:** Cross-origin request blocked
**Solution:**
1. Firebase Cloud Function: Already handled, no CORS issues
2. Custom API: Add CORS headers: `Access-Control-Allow-Origin: *`

### Maximum retries exceeded
**Problem:** Email failed 3 times
**Solution:**
1. Check email service status
2. Verify credentials
3. Check pending emails: `JSON.parse(localStorage.getItem('pendingOrderEmails'))`
4. Manually retry: `EmailService.retryPendingEmails()`

## Email Template Customization

The HTML template is generated in [js/email.js](js/email.js) function `generateOrderConfirmationHTML()`.

To customize:

1. Colors: Change `#667eea` and `#764ba2` to your brand colors

3. Content: Modify text, add promotional messages, etc.

## Advanced Features

### Retry Failed Emails

Emails that fail are automatically stored and can be retried:

```javascript
// Retry all pending emails
await EmailService.retryPendingEmails();

// Check pending emails
const pending = JSON.parse(localStorage.getItem('pendingOrderEmails') || '[]');
console.log(`${pending.length} emails pending`);
```

### Send Additional Emails

Extend EmailService for shipment emails, refunds, etc.:

```javascript
async sendShipmentEmail(order, user, trackingNumber) {
  // Generate shipment email HTML
  // Send via active service
}
```
